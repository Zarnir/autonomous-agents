The problem

AI coding tools are great at one-shot tasks. They break down on multi-step work.

Hallucination

"Implement X with library Y" → uses functions that don't exist; tests never written; PR looks fine.

Scope drift

"Add login" → rewrites your entire auth stack; touches 40 files; you can't review the diff.

No tests

Claims "done" — but tests were never run. CI catches it three commits later.

Lost state

You quit the session; tomorrow you're starting over. The agent has no memory of yesterday.

The problem isn't the LLM — it's the lack of structure around it: planning, scope enforcement, real test verification, durable state, retries that don't waste budget.

The solution

A deterministic Python orchestrator driving LLM agents per role.

SCRUM-style sprints. Real TDD enforcement (failing test BEFORE code). Resumable state machine. Every commit is gated on independent test re-run.

Plans

One-line idea → epics → stories → tasks → acceptance criteria. Deterministic parser, no LLM hallucination.

Executes

Story by story, 6 specialized agents per story. Isolated git worktree per story. Scope enforced by @guard.

Verifies

Orchestrator runs the tests itself — never trusts the agent's "GREEN" claim. Production gates block bad sprints.

Architecture · story lifecycle

Each story is a finite state machine.

Every story passes through these states. Persisted on disk after every transition — so any kill mid-flight resumes cleanly.

Happy path
pending — run_story → in_progress — @check + @simplify PASS → review_pass — @test GREEN → test_written — @make + tests GREEN → implemented — @guard PASS + @commit → completed
Failure paths
any in-flight state — AgentError → failed

Hard agent failure (timeout, model error, contract violation after retries).

pending — upstream fail → blocked

cascade_fail propagates upstream failure to all dependents.

Recovery
resume

Stuck in in_progress/review_pass/test_written → reset to pending; re-run from last clean checkpoint.

resume --retry-failed

All failed stories → back to pending.

resume --retry-blocked

All blocked stories → back to pending.

Per-story timing recorded (M24): started_at · completed_at · failure_reason — all surfaced in aa-orchestrator status.

Safety

Six guarantees the orchestrator enforces.

Not promises — actual code that fails the pipeline if violated.

Tests are real

Orchestrator re-runs the test suite itself after @make claims GREEN. Never trusts the agent.

Scope is enforced

@guard audits git diff against declared files_to_touch. Out-of-scope writes trigger retry or fail.

Every AC has a test

validate_criterion_coverage rejects RED_VERIFIED if any acceptance criterion isn't mapped to a test.

State survives

Every transition fsync'd before next phase. aa-orchestrator resume picks up cleanly after any kill.

Budget enforced

MAX_BUDGET_USD halts mid-call when exceeded. Per-agent cost tracking in progress.json.

Production gates

Clean working tree + all tests pass + build succeeds — before a sprint is considered complete.

Get started

Four commands. Then walk away.

# 1. One-time machine setup
$ git clone <repo> ~/code/autonomous-agents
$ cd ~/code/autonomous-agents
$ bash install.sh

# 2. Bootstrap a new project
$ aa-orchestrator new my-app --interactive

# 3. Build the execution plan (no LLM cost)
$ cd my-app
$ aa-orchestrator develop --dry-run

# 4. Run the whole thing autonomously
$ aa-orchestrator sprint cycle
Step 1 installs

Global CLI at ~/.local/bin/aa-orchestrator + agents + skills + slash commands.

Step 2 asks

"Which LLM runner?" · "Got an idea?" · then auto-generates a SCRUM spec tree.

Step 3 plans

Deterministic parse + plan. ~1 second. Free. Writes progress.json.

Step 4 builds

Plan → execute → retro → groom → next sprint. One git commit per story. Until the backlog is empty.

Or drive from inside chat: cd my-app && claude, then /sprint cycle.
